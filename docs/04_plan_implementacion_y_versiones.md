# Plan de Implementación y Versiones

## Objetivo

Definir el orden correcto de trabajo para no reabrir caos en el alcance y para alinear el desarrollo con el plan general del proyecto.

## Versión actual cerrada parcialmente
- flujo Excel,
- modo inicial,
- validación de entrada,
- corrida PSO,
- historial,
- detalle,
- exportación,
- estados básicos,
- persistencia local base.

## Próximas fases / versiones

### V1.0 — Base operativa
- flujo Excel,
- motor integrado,
- base de datos de corridas,
- docs base y checkpoint compartible.

### V1.1 — Interfaz corporativa Windows
- menús superiores,
- toolbar,
- navegación clásica,
- formularios y tablas más empresariales.

### V1.2 — Acceso y perfiles
- usuarios locales,
- login,
- perfil Operador,
- perfil Ingeniero / Programador,
- permisos de edición.

### V1.3 — Configuración editable
- parámetros editables,
- base de datos de configuración,
- restaurar valores por defecto,
- snapshot por corrida,
- evaluación de servidor propio.

### V1.4 — Carga manual y gestión de corridas
- formularios manuales,
- validación manual,
- comparación entre corridas,
- favoritos / caso base,
- borrado automático mensual.

### V1.5 — Validación del modelo y reprogramación
- benchmark,
- tolerancias,
- criterios de aceptación,
- alcance e implementación de reprogramación.

### V1.6 — Integraciones, despliegue y cierre
- scraping COES,
- alertas,
- empaquetado,
- distribución manual,
- actualización manual,
- pruebas en equipo limpio,
- transferencia y capacitación.

## Regla de control

No abrir una versión nueva sin documentar la anterior y sin dejar claro qué cambia funcional y técnicamente.
